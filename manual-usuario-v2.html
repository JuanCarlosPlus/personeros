7 Robot WhatsApp (Servicio Interno)

El robot de WhatsApp consume la API para procesar mensajes pendientes. No tiene interfaz grafica propia — opera mediante endpoints REST.

📡 Endpoints del Robot
MetodoEndpointDescripcion
GET/api/v1/whatsapp/colaObtener cola de mensajes pendientes
PATCH/api/v1/whatsapp/:id/enviadoMarcar mensaje como enviado exitosamente
PATCH/api/v1/whatsapp/:id/errorMarcar mensaje como error de envio

10 Flujos Principales del Sistema

Resumen de los procesos clave del sistema.

🔄 Flujo 1: Registro por Invitacion
Admin crea Directivo
→
Directivo invita (WhatsApp)
→
Bot envia link
→
Invitado se registra
→
Personero creado
🔄 Flujo 2: Registro por Jefe de Local
Admin crea Jefe Local
→
Jefe inicia sesion (OTP)
→
Busca DNI (Factiliza)
→
Registra personero
→
Asigna a mesa
🔄 Flujo 3: Registro Masivo (Admin)
Admin abre Registro Masivo
→
DNI + Enter (API Factiliza)
→
Datos auto-completados
→
Repite N veces
→
Guardar todos
🔄 Flujo 4: Login Jefe de Local (2 pasos)
Ingresa telefono
→
Bot envia OTP por WhatsApp
→
Ingresa codigo OTP
→
Acceso al dashboard del local
